# 移动端（学生/教练端）首页重构设计规范

## 1. 核心设计方向：任务驱动型日程流（方式 B3）
针对教练端与学生端，采用“顶部水平日期条 + 下方纵向日程流”的布局。强化当前及未来一周的任务/课程安排，弱化传统九宫格的跳转。

## 2. 界面结构规范
### 2.1 顶部数据概览卡片 (Top Summary Card)
- **教练端**：极简数据栏，显示今日上课总人数、待签到人数。
- **学生端**：极简数据栏，显示剩余总课时、最近一次体测得分。
- **视觉风格**：深色/主题色底色白字，带有圆角，高度压缩，不喧宾夺主。

### 2.2 水平日期选择器 (Horizontal Date Bar)
- **交互**：左右滑动选择日期，默认选中高亮“今日”。
- **状态提示**：日期下方带有红点（有课）或灰点（休息）。
- **扩展**：提供一个日历小图标，点击可展开全月视图。

### 2.3 纵向日程卡片流 (Vertical Schedule Stream)
- 按照选中日期的时间顺序从上到下排列课程卡片。
- **卡片构成**：
  - 左侧：醒目的大字号上课时间（如 14:00 - 15:30）。
  - 右上：课程名称与状态标签（未开始、进行中、已结课）。
  - 右中：教练端显示上课名单/场地；学生端显示教练/场地。
  - 右下/底部操作区：快捷操作按钮（实心按钮为推荐操作，空心为次要操作）。

### 2.4 快捷操作按钮规范
- **教练端卡片操作**：点名签到（主）、录入体测（主）、请假审批。
- **学生端卡片操作**：请假（次）、查看详情。

### 2.5 全局悬浮操作 (FAB - 仅教练端)
- 右下角常驻悬浮按钮（FAB），提供最高频全局操作。
- 建议：扫码签到 或 +排课。

## 3. 视觉与反馈规范
- **留白与间距**：卡片间距 24rpx，内部间距 20rpx，四周 padding 32rpx。
- **触控反馈**：所有卡片和列表项增加按压阴影或变色反馈（:active 态）。
- **空状态**：当选中的日期没有课程时，展示精美的缺省图（如“今天没有安排，好好休息吧”）。

## 4. 技术实现建议 (基于 uview-plus)
- 日期滑动条可基于 scroll-view 或 u-tabs 自定义改造。
- 课程卡片使用 Flex 布局。
- 页面整体下拉刷新更新当前日期数据。
